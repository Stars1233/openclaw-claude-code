# Claude Code CLI 2.1.111 Feature Update

**Date:** 2026-04-16
**Target version:** 2.13.0 (minor — new features, backward compatible)
**Claude Code CLI:** 2.1.91 → 2.1.111

## Scope

Add all Tier 1 + Tier 2 features from Claude Code CLI versions 2.1.92–2.1.111 to the openclaw-claude-code plugin. Tier 3 (human-interactive-only features) are excluded.

## Changes

### 1. New SessionConfig fields (`src/types.ts`)

Add to `SessionConfig` interface:

| Field | Type | CLI Flag | Purpose |
|-------|------|----------|---------|
| `includeHookEvents` | `boolean` | `--include-hook-events` | Stream hook lifecycle events (PreToolUse/PostToolUse) |
| `permissionPromptTool` | `string` | `--permission-prompt-tool <tool>` | Delegate permission prompts to MCP tool for non-interactive use |
| `excludeDynamicSystemPromptSections` | `boolean` | `--exclude-dynamic-system-prompt-sections` | Move cwd/env/git from system prompt to user message for better cache hits |
| `debug` | `string \| string[]` | `--debug <categories>` | Enable debug output for specific categories (e.g. "api,mcp") |
| `debugFile` | `string` | `--debug-file <path>` | Write debug output to file instead of stderr |
| `fromPr` | `string` | `--from-pr <number\|url>` | Resume session linked to a GitHub PR |
| `channels` | `string \| string[]` | `--channels <spec>` | MCP channel subscriptions (research preview) |
| `dangerouslyLoadDevelopmentChannels` | `string \| string[]` | `--dangerously-load-development-channels <spec>` | Load dev MCP channels |
| `enablePromptCaching1H` | `boolean` | env: `ENABLE_PROMPT_CACHING_1H=1` | 1-hour prompt cache TTL (vs default 5-min) |

### 2. Smart defaults

When `bare: true` and user has NOT explicitly set `excludeDynamicSystemPromptSections: false`:
- Auto-enable `--exclude-dynamic-system-prompt-sections`

Rationale: `bare` users want isolation and speed. Dynamic sections (cwd, env, git status) in system prompt bust the prompt cache on every different directory. This is a natural pairing.

When `enablePromptCaching1H: true` OR (`bare: true` and user has NOT explicitly set it to `false`):
- Pass `ENABLE_PROMPT_CACHING_1H=1` to the child process env

### 3. CLI flag construction (`src/persistent-session.ts` → `start()`)

Add to the args-building section (after existing "New CLI flags" block around line 193):

```
if (this.options.includeHookEvents) args.push('--include-hook-events');
if (this.options.permissionPromptTool) args.push('--permission-prompt-tool', this.options.permissionPromptTool);
if (shouldExcludeDynamicSections) args.push('--exclude-dynamic-system-prompt-sections');
if (this.options.debug) {
  const cats = Array.isArray(this.options.debug) ? this.options.debug.join(',') : this.options.debug;
  args.push('--debug', cats);
}
if (this.options.debugFile) args.push('--debug-file', this.options.debugFile);
if (this.options.fromPr) args.push('--from-pr', this.options.fromPr);
if (this.options.channels) {
  const ch = Array.isArray(this.options.channels) ? this.options.channels : [this.options.channels];
  for (const c of ch) args.push('--channels', c);
}
if (this.options.dangerouslyLoadDevelopmentChannels) {
  const ch = Array.isArray(this.options.dangerouslyLoadDevelopmentChannels)
    ? this.options.dangerouslyLoadDevelopmentChannels
    : [this.options.dangerouslyLoadDevelopmentChannels];
  for (const c of ch) args.push('--dangerously-load-development-channels', c);
}
```

Env var handling (in the `spawn` call's env):
```
if (this.options.enablePromptCaching1H || (this.options.bare && this.options.enablePromptCaching1H !== false)) {
  env.ENABLE_PROMPT_CACHING_1H = '1';
}
```

### 4. New event handling (`src/persistent-session.ts` → `_handleEvent()`)

Add a case for `system` events with retry subtype. The event shape:

```json
{
  "type": "system",
  "subtype": "api_retry",
  "attempt": 2,
  "max_retries": 5,
  "retry_delay_ms": 2000,
  "error_status": 529,
  "error_category": "overloaded"
}
```

Handling:
- Increment `this.stats.retries`
- Store `lastRetryError` = `error_category` or `error_status`
- Forward through `callbacks.onEvent()` so callers can react

### 5. Stats extension (`src/types.ts`)

Add to whatever stats type tracks session metrics:
```typescript
retries: number;         // total API retry attempts
lastRetryError?: string; // last retry error category
```

Initialize `retries: 0` in stats initialization.
